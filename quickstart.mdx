---
title: "Quickstart"
description: "Get your first Foreman development container running on OrbStack in a few commands."
---

<Note>
  foreman-builder requires **Apple Silicon (arm64)**. It will not run on Intel-based Macs.
</Note>

<Steps>
  <Step title="Verify prerequisites">
    Before you begin, confirm the following are in place:

    - You are on a Mac with Apple Silicon (M1 or later).
    - [OrbStack](https://orbstack.dev/) is installed and running. The `orbctl` binary must be in your `PATH`.
    - [Go](https://go.dev/doc/install) is installed (version 1.21 or later recommended).
    - You have network access — setup downloads Ruby, Node.js, and the forklift repository.

    Check that OrbStack is running:

    ```bash
    orbctl status
    ```

    Check that Go is available:

    ```bash
    go version
    ```
  </Step>

  <Step title="Install foreman-builder">
    Install the latest version directly from source using `go install`:

    ```bash
    go install github.com/aidenfine/foreman-builder@latest
    ```

    The binary is placed in `$(go env GOPATH)/bin`. Make sure that directory is in your `PATH`. Verify the installation:

    ```bash
    foreman-builder --help
    ```
  </Step>

  <Step title="Create your first container">
    Run the `create` command to spin up a new Foreman development container:

    ```bash
    foreman-builder create
    ```

    You will be prompted to enter a container name. Press **Enter** to accept the default name `foreman`, or type a custom name.

    foreman-builder reads an optional `./config.yml` file from your current directory if it exists. You can use this to specify additional packages to install inside the container. If no config file is present, the default setup runs without modification.
  </Step>

  <Step title="Wait for cloud-init to finish">
    After the container is created, OrbStack starts a cloud-init process inside it. This process:

    - Installs system dependencies
    - Sets up rbenv and Ruby 3.1.6
    - Installs Node.js 22
    - Installs Ansible
    - Clones the [forklift](https://github.com/theforeman/forklift) repository
    - Runs the `katello_devel.yml` Ansible playbook

    <Warning>
      This process takes a significant amount of time — typically 20 to 40 minutes depending on your network speed. The `foreman-builder create` command returns as soon as the container is created, not when cloud-init completes.
    </Warning>

    To follow progress, stream the cloud-init log from inside the container:

    ```bash
    ssh foreman@orb cat /var/log/cloud-init-output.log
    ```

    Re-run the command periodically to check whether the playbook has finished.
  </Step>

  <Step title="SSH into the container">
    OrbStack provides direct SSH access to containers using the `@orb` hostname suffix:

    ```bash
    ssh foreman@orb
    ```

    Replace `foreman` with your container name if you used a custom one. No additional SSH key configuration is required — OrbStack handles authentication automatically.
  </Step>

  <Step title="Open Foreman in your browser">
    OrbStack forwards ports from the container to `localhost` transparently. Once cloud-init has finished and Foreman services are running, open your browser and navigate to:

    ```
    http://localhost:3000
    ```

    <Tip>
      The default Foreman development credentials are `admin` / `changeme`. Refer to the forklift documentation for details on changing these.
    </Tip>
  </Step>
</Steps>
